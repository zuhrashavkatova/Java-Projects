Auto Service

Develop a Java application, which automates Auto service system. Auto service company receives from customer an order to repair the car. You need to develop the system which allows to organize this process and find nearby Auto services for customer. All the classes must be in the package "uz.ttpu".


R1 - AutoService

The system works through the facade class CarRepairing. 

The method createAutoService() accepts the auto service id, name and its coordinate (X and Y axes) and creates a AutoService object. The class AutoService provides the method getName(), which returns the AutoService’s name.

The method getAutoServices() returns a Collections of AutoServices. The method getAutoService() receives the auto service id as an input and returns the corresponding Auto service object. In case if auto service with corresponding id does not exist, exception NoSuchAutoService is thrown.

By default we assume that auto services work with any kind of services


R2 - Driver

Drivers are added by administrator. The method addDriver() is used to add driver by means of ID, full name and car brand. In case if driver with corresponding ID exist, exception DriverAlreadyExist is thrown. The class Driver offers the method getName(), which returns the Employees full name.

To obtain a specific Driver it is possible to use the method getDriver(), which receives the ID of the Driver as a parameter. In case if driver with corresponding ID does not exist, exception NoSuchDriver is thrown.

The method getDrivers() returns a Collections of Drivers.



R3 - Services

Auto service provides different kind of services (oil change, body repair, diagnostics, battery replacement...). Each service needs to be created by administrator before it is used by driver or auto service.

The method createService() accepts the service id and name and its cost and creates a Service object.

The method getService() receives the service id as an input and returns the corresponding service object. In case if auto service with corresponding id does not exist, exception NoSuchService is thrown.



R4 - Booking

An auto service has method availableAutoServices() which accepts driver ID, driver current position coordinate (X and Y axes) and list of all services needs to be handled. Method returns list of 3 auto services which are located nearby and sorted in ascending way by distance from drivers coordinate to auto services coordinate.

An auto service could be booked by means of driver ID, autoService ID and list of all services needs to be handled. This operation could be done through the method bookAutoService().

Method findDistanceBetweenTwoCoordinates() is used to find distance between two coordinates. Method gets two coordinates as an input and returns distance between two coordinates and returns as an output double.



R5  - Statistics

Given an auto driver ID, it is possible to know how many times drivers have booked by means of method: getNumberOfBookingsByDrivers().

Given an auto service ID, it is possible to calculate total profit of auto service by method: getAutoServiceTotalProfit().
